    Solución

    1. Un lenguaje de marcas es un sistema de codificación que utiliza etiquetas o marcas para estructurar y organizar el contenido de un documento, proporcionando información adicional sobre el significado y la relación entre los elementos. Es relevante en el desarrollo web porque permite definir la estructura y el contenido de una página web de manera coherente y legible para los humanos y las máquinas.

      Tres características clave de los lenguajes de marcas son:

      • Estructura jerárquica: Los elementos se organizan en una estructura de árbol, donde cada elemento puede tener uno o varios elementos secundarios. Etiquetas: Se utilizan para marcar el inicio y el final de un elemento, indicando su tipo y contenido.
      • Interoperabilidad: Al seguir estándares comunes, como HTML o XML, se garantiza la compatibilidad entre diferentes sistemas y plataformas. Un ejemplo concreto de un lenguaje de marcas utilizado en la web es HTML (HyperText Markup Language), que se utiliza para definir la estructura y el contenido de las páginas web .

    2. La diferencia entre una etiqueta de apertura y una etiqueta de cierre en un lenguaje de marcas es que la etiqueta de apertura marca el inicio de un elemento y la etiqueta de cierre marca el final del mismo. Por ejemplo, en HTML,

    3. Es importante que un documento esté bien formado para garantizar su correcta interpretación por parte de los programas informáticos. Un documento se considera válido cuando cumple con todas las reglas sintácticas y estructurales del lenguaje de marcas utilizado, como tener etiquetas de apertura y cierre correspondientes y una estructura jerárquica adecuada.

    4. La evolución de los lenguajes de marcas desde GML hasta XML ha sido un proceso de avances significativos en la estructura y funcionalidad. Algunos hitos importantes incluyen la creación de HTML como estándar para la creación de páginas web, que simplificó la creación de contenido web, y la introducción de XML como un lenguaje de marcas puramente estructural, independiente del diseño.

    5. HTML es importante como estándar para la creación de páginas web porque proporciona una forma común de estructurar y presentar contenido en la web. Sin embargo, tiene limitaciones en cuanto a flexibilidad y estructura, ya que está más centrado en la presentación que en el intercambio de datos. XML aborda estas deficiencias al proporcionar una estructura más flexible y extensible, lo que lo hace más adecuado para el intercambio de datos entre sistemas heterogéneos.

    6. Un XML Namespace es un mecanismo que permite evitar conflictos de nombres al definir etiquetas personalizadas en un documento XML. Su propósito es proporcionar un contexto único para las etiquetas, lo que permite distinguir entre elementos con nombres similares pero definidos en diferentes vocabularios o namespaces. Por ejemplo, en un documento XML que contiene elementos de diferentes vocabularios, se pueden utilizar namespaces para garantizar que no haya ambigüedad en la interpretación de las etiquetas por parte de un software o una máquina.

    7. DTD significa Document Type Definition y su función en un documento XML es definir la estructura y las reglas que deben cumplir los elementos y atributos del documento. Se utiliza para validar documentos XML y garantizar su conformidad con las reglas especificadas. Por ejemplo, un DTD puede especificar que un documento XML debe contener ciertos elementos en un orden específico y con ciertos atributos.

    8. Tres lenguajes de marcas de tres ámbitos de aplicación son:

      • HTML: Utilizado para la creación de páginas web.
      • SVG (Scalable Vector Graphics): Utilizado para gráficos vectoriales escalables en la web.
      • MathML (Mathematical Markup Language): Utilizado para representar fórmulas matemáticas en documentos web. Estos lenguajes surgieron para satisfacer necesidades específicas en cada ámbito, ya que tienen estructuras y características adaptadas a los requisitos únicos de cada aplicación. Por ejemplo, SVG se enfoca en gráficos vectoriales, mientras que MathML se centra en representar fórmulas matemáticas de manera precisa
