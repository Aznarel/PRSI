    Actividades refuerzo¶

    Las actividades de refuerzo son tareas diseñadas para ayudar al alumnado que no ha consolidado los conocimientos y/o habilidades trabajadas en la unidad de trabajo mediante las actividades evaluables de aula. Se caracterizan por facilitar más tiempo para la comprensión y abordar los objetivos mediante distintos estilos de aprendizaje asegurando que se llega a todo el alumnado de una forma eficiente.

    AAR1.6 Tabla resumen¶

    (RA1 / CE1a, CE1b, CE1c, CE1d, CE1e, CE1f, CE1g, CE1h, CE1i / IC1-3p)

    Nuestro jefe de departamentos nos pide crear una una tabla resumen de los siguientes conceptos:

    1. Explica con tus palabras, qué ventajas proporcionan los lenguajes de marcas en el tratamiento de la información
    2. ¿Qué es la W3C? ¿Cuáles son sus objetivos?
    3. Clasifica los lenguajes de marcas e identificado los más relevantes divididos por ámbito de aplicación.
    4. ¿Por qué hay documentos web que no se visualizan correctamente en algunos navegadores?
    5. Características del metalenguaje: Aquí puedes explicar brevemente qué significa que XML es un metalenguaje, es decir, un lenguaje para definir otros lenguajes de marcado.
    6. ¿Qué es un Documento bien formado? Describe las reglas que debe cumplir un documento XML para considerarse bien formado.
    7. ¿Qué estructura debe tener un Documento bien formado? Detalla la estructura básica de un documento XML.
    8. ¿Qué son los espacios de nombres en XML y por qué son importantes en la creación de documentos XML? Define qué son los espacios de nombres en XML y cómo permiten evitar conflictos entre diferentes vocabularios XML utilizados en un mismo documento. Explica su importancia en la reutilización de elementos y atributos.

    Consejos para la realización:

    • Utiliza ejemplos sencillos para ilustrar cada concepto.
    • Definiciones claras y concisas.

    AAR1.7 Software I¶

    (RA1 / CE1f, CE1g, CE1i /IC-3p)

    Se desea crear un formato de intercambio de datos para una empresa mayorista de software con el fin de que sus distintos programas puedan manejar la información interna.

    Nos piden crear un documento XML que represente un catálogo de software informático. Utiliza espacios de nombres para diferenciar entre software de sistemas operativos y software de aplicaciones de oficina. Asegúrate de que el documento está bien formado y tiene la siguiente estructura:

    1. Nombre del Archivo: softwareCatalogYourName.xml (cambia YourName por tu nombre completo)
    2. Estructura del Documento:
      • Declaración XML estándar al inicio.
      • Elemento raíz: <catalogo>.
      • Dos espacios de nombres:
        • os: Para software de sistemas operativos.
        • office: Para software de aplicaciones de oficina.
      • Incluye al menos tres elementos para cada tipo de software.
      • Cada elemento de software debe contener:
        • nombre: Nombre del software.
        • version: Versión actual del software.
        • desarrollador: Nombre del desarrollador o compañía.
        • url: URL para más información (opcional)
        • otro: Característica específica de os u office según el caso
    Solución 
    <?xml version="1.0" encoding="UTF-8"?>
<catalogo xmlns:os="http://www.ejemplo.com/namespace/os" xmlns:office="http://www.ejemplo.com/namespace/office">

<!-- Software de Sistemas Operativos -->
<os:software>
    <os:nombre>Windows</os:nombre>
    <os:version>10</os:version>
    <os:desarrollador>Microsoft</os:desarrollador>
    <os:url>https://www.microsoft.com/windows</os:url>
    <os:otro>Interfaz gráfica</os:otro>
</os:software>

<os:software>
    <os:nombre>Ubuntu</os:nombre>
    <os:version>20.04 LTS</os:version>
    <os:desarrollador>Canonical</os:desarrollador>
    <os:url>https://ubuntu.com/</os:url>
    <os:otro>Kernel Linux</os:otro>
</os:software>

<os:software>
    <os:nombre>macOS</os:nombre>
    <os:version>Big Sur</os:version>
    <os:desarrollador>Apple</os:desarrollador>
    <os:url>https://www.apple.com/macos/</os:url>
    <os:otro>Integración con hardware Apple</os:otro>
</os:software>

<!-- Software de Aplicaciones de Oficina -->
<office:software>
    <office:nombre>Microsoft Office</office:nombre>
    <office:version>365</office:version>
    <office:desarrollador>Microsoft</office:desarrollador>
    <office:url>https://www.microsoft.com/office/</office:url>
    <office:otro>Suite completa de productividad</office:otro>
</office:software>

<office:software>
    <office:nombre>LibreOffice</office:nombre>
    <office:version>7.2</office:version>
    <office:desarrollador>The Document Foundation</office:desarrollador>
    <office:url>https://www.libreoffice.org/</office:url>
    <office:otro>Software de código abierto</office:otro>
</office:software>

<office:software>
    <office:nombre>Google Workspace</office:nombre>
    <office:version>Enterprise</office:version>
    <office:desarrollador>Google</office:desarrollador>
    <!-- No hay URL proporcionada para este ejemplo -->
    <office:otro>Colaboración en línea</office:otro>
</office:software>

</catalogo>
